# Agent1 — Text-to-SQL Crop Yield Agent
 
A **LangGraph-based agent** that converts natural language questions into SQL queries, runs them against a crop yield database, and returns human-readable insights. Deployable locally via Flask or on a **Kagenti cluster** via the A2A protocol.
 
---
 
## Project Structure
 
```
agent1/
├── app.py               # Flask + A2A endpoints
├── graph.py             # LangGraph agent logic (3-node pipeline)
├── data.py              # One-time DB setup script
├── crop_yield.csv       # Source data
├── crop.db              # SQLite database (generated by data.py)
├── requirements.txt     # Python dependencies
├── Dockerfile           # Container build config
└── k8s/
    ├── deployment.yaml
    ├── service.yaml
    ├── serviceaccount.yaml
    └── agentcard.yaml
```

## How It Works
 
### `data.py`
> Run **once** to generate `crop.db` from `crop_yield.csv`. Do not re-run unless the CSV has been updated.
 
### `graph.py`
Contains the core agent logic using an `AgentState` dictionary and a 3-node LangGraph pipeline:
 
| Step | Node | Description |
|------|------|-------------|
| 1 | **NL → SQL** | Converts the user's natural language question into a SQL query |
| 2 | **Run Query** | Executes the SQL query against `crop.db` |
| 3 | **SQL → NL** | Translates the query results back into a natural language insight |
 
The agent follows a single linear path. On completion, `AgentState` holds the generated query, raw data rows, and final insights.
 
### `app.py`
Exposes two routes:
 
- **Flask (local)** — Invokes the agent and returns SQL results + insights as HTML for browser-based interaction
- **A2A (Kagenti)** — Invokes the agent and returns results as JSON via the A2A protocol, which the Kagenti backend converts to text for its chat UI

## Prerequisites
 
- Python (version per `requirements.txt`)

- Docker Desktop
- `kubectl` + `kind`

## Getting Started
 
### 1. Install Dependencies
 
```bash
pip install -r requirements.txt
```
 
> You may need to reinstall Python within your virtual environment to meet the version requirements.
 
---
 
### 2a. Run Locally (Flask)
 
```bash
# Step 1: Generate the database (first time only)
python data.py
 
# Step 2: Start the Flask app
python app.py
```
 
Then open `http://localhost:{PORT}` in your browser to interact with the UI.
 
> **Note:** Check the `Dockerfile` and `app.py` for the port — update `0.0.0.0:{PORT}` if another process is already using it.
 
---
 
### 2b. Deploy on Kagenti (A2A Protocol)
 
#### Install Kagenti
See the [Kagenti GitHub](https://github.com) for installation instructions.
 
#### Apply Kubernetes Manifests
 
```bash
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/serviceaccount.yaml
kubectl apply -f k8s/agentcard.yaml
```
 
#### Build & Push Docker Image
 
**Option A:**
```bash
docker build -t crop-yield-agent:latest .
kind load docker-image crop-yield-agent:latest --name kagenti
kubectl set image deployment/crop-yield-agent <name_of_agent>=crop-yield-agent:latest -n <namespace>
kubectl rollout restart deployment/crop-yield-agent -n <namespace>
```
 
**Option B:**
```bash
docker build -t 127.0.0.1:5000/crop-yield-agent:latest .
kind load docker-image 127.0.0.1:5000/crop-yield-agent:latest --name kagenti
kubectl rollout restart deployment/crop-yield-agent -n <namespace>
kubectl rollout status deployment/crop-yield-agent -n <namespace> --timeout=60s
```
 
#### Verify Pods Are Running
 
```bash
kubectl get pods -n <namespace>
```
 
#### Access the UI
Log in to the **Kagenti UI** with your admin credentials and interact with the agent via the chat interface.
 
---
 
## Notes
 
- `data.py` only needs to be run once (or when `crop_yield.csv` is updated)
- If port conflicts occur, update the port in both `Dockerfile` and `app.py`
- Replace `<namespace>` and `<name_of_agent>` with your actual Kubernetes values throughout the deploy commands
